# Glossary

Common terms used across the team. Keep this updated as new terms emerge.

| Term | Definition |
|------|-----------|
| **Skill** | A Claude Code Skill defined by a SKILL.md file — represents one AI team member |
| **SKILL.md** | The definition file for a Skill, containing YAML frontmatter (trigger conditions) and Markdown body (work instructions) |
| **Department** | A logical grouping of related Skills (e.g., engineering, marketing) |
| **Trigger** | The condition (keyword match in user input) that activates a Skill |
| **References** | Supporting documents that a Skill can load on demand for additional context |
| **Orchestrator** | A meta-Skill (like a CTO) that decomposes tasks and delegates to other Skills |
| **Three-layer loading** | Context management: Layer 1 (name+description, always loaded), Layer 2 (SKILL.md body, on trigger), Layer 3 (references, on demand) |
| **PRD** | Product Requirements Document |
| **CTA** | Call to Action — a prompt for the reader/user to take a specific next step |
| **MECE** | Mutually Exclusive, Collectively Exhaustive — a principle for thorough, non-overlapping breakdowns |
| **MVP** | Minimum Viable Product — the smallest version that delivers core value |
| **STAR** | Situation, Task, Action, Result — a framework for behavioral interview answers |
| **WCAG** | Web Content Accessibility Guidelines |
