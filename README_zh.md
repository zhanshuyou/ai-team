# AI Team — 用 Claude Code 搭建你的 AI 虚拟团队

用 Claude Code 的 Skill 机制，把 AI 组织成一支**有分工、能协作、可编排**的虚拟团队。

## 这是什么

一套实践指南，教你如何：

- 把不同职能的 AI 定义为"员工"（产品经理、前端开发、代码审查员、数据分析师……）
- 让员工之间自动协作，串联复杂任务
- 将 AI 团队无缝集成到你的现有项目中

每个"员工"本质上是一个 **Skill**，由 `SKILL.md`（岗位说明书）+ 辅助资源（参考文档、脚本、模板）组成。

## 核心概念

### Skill = 员工

```
departments/engineering/frontend-dev/
├── SKILL.md              # YAML 头部（触发条件）+ Markdown 正文（工作指南）
├── references/           # 参考文档（编码规范、模板等）
├── scripts/              # 自动化脚本
└── assets/               # 模板文件
```

### 三层上下文加载

| 层级 | 内容 | 何时加载 |
|------|------|----------|
| 第 1 层 | name + description | 始终在上下文中 |
| 第 2 层 | SKILL.md 正文 | 触发时加载 |
| 第 3 层 | references / scripts | 需要时按指引加载 |

### 编排方式

从对话式到全自动，三种方案可组合使用：

1. **编排者 Skill**（CTO）— 拆解任务、分配员工、串联流程
2. **CLAUDE.md 全局规则** — 定义公司规则和员工清单，启动时自动生效
3. **脚本驱动** — `run-workflow.sh` 一条命令启动流水线

## 快速开始

1. 选择一份指南开始阅读
2. 创建你的第一个员工 `SKILL.md`
3. 用一个小任务测试效果（比如"写一个按钮组件"）
4. 根据效果迭代调整，逐步添加更多员工
